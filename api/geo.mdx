The `Geo` class (in `helpers/Geo.py`) accepts a pandas `DatetimeIndex` and site coordinates, then populates a DataFrame with all relevant solar geometry columns needed for quality control and clear-sky modeling. On construction every column is computed in a single pass over the date range ŌĆö no further method calls are required to access the results.

---

## Constructor

```python
Geo(range_dates, lat, long, gmt, alt, beta)
```

### Parameters

<ParamField path="range_dates" type="pd.DatetimeIndex" required>
  Date range for which to compute geometry. Typically generated with `pd.date_range(start=..., end=..., freq=...)`.
</ParamField>

<ParamField path="lat" type="float" required>
  Site latitude in decimal degrees. Negative values indicate the Southern Hemisphere (e.g. `-22.1` for north-western Argentina).
</ParamField>

<ParamField path="long" type="float" required>
  Site longitude in decimal degrees. Negative values indicate west of the Prime Meridian.
</ParamField>

<ParamField path="gmt" type="int" required>
  UTC offset of the site (e.g. `0` for UTC, `-3` for Argentina standard time). The constructor uses `abs(gmt)` internally when computing the sign correction for solar hour.
</ParamField>

<ParamField path="alt" type="float" required>
  Site altitude in metres above sea level. Used for the air-mass pressure correction and for choosing the `ktrp` branch in the ARGP clear-sky model.
</ParamField>

<ParamField path="beta" type="float" required>
  Panel tilt angle in degrees. Pass `0` for a horizontal surface (GHI geometry).
</ParamField>

### Returns

An instance whose `.df` attribute is a `pd.DataFrame` containing all computed columns described in the table below. The constructor has no additional return value.

---

## Output DataFrame: `Geo.df`

Every row corresponds to one timestamp in `range_dates`. All columns are appended in the order listed.

| Column | Type | Description |
|--------|------|-------------|
| `date` | datetime | Input timestamp from `range_dates` |
| `N` | int | Day of year (1ŌĆō366) |
| `M` | int | Month (1ŌĆō12) |
| `Y` | int | Four-digit year |
| `E` | float | Equation of time in minutes |
| `HR` | float | Local clock hour as a decimal (e.g. `13.5` = 13:30) |
| `HS` | float | Solar hour as a decimal, corrected for longitude and equation of time |
| `deltaRad` | float | Solar declination in radians |
| `delta` | float | Solar declination in degrees |
| `ws` | float | Sunset hour angle in radians: `arccos(ŌłÆtan(╬┤)┬Ętan(lat))` |
| `Fn` | float | Orbital distance correction factor (Fourier series) |
| `w` | float | Hour angle in degrees: `15 ├Ś (12 ŌłÆ HS)` |
| `wRad` | float | Hour angle in radians |
| `lat` | float | Site latitude (constant column, repeated for every row) |
| `CTZ` | float | `cos(╬Ėz)` ŌĆö cosine of the solar zenith angle |
| `TZ` | float | Solar zenith angle `╬Ėz` in radians: `arccos(CTZ)` |
| `SZA` | float | Solar Zenith Angle in degrees |
| `alphaS` | float | Solar elevation angle in degrees: `arcsin(CTZ)` in degrees |
| `E0` | float | Orbital eccentricity correction factor |
| `TOA` | float | Top-of-atmosphere irradiance in W/m┬▓; `0` when `CTZ < 0` |
| `Mak` | float | Kasten air mass (pressure-corrected for site altitude) |
| `GHIargp` | float | Clear-sky GHI from the ARGP model using `ktrp1` (altitude Ōēż 1000 m formula or > 1000 m formula, see `getKtrp`) |
| `GHIargp2` | float | Clear-sky GHI from the ARGP model using `ktrp2 = 0.649 + 0.02┬Ęalt^0.28` |
| `mr` | float | Relative air mass (Reindl formula, pressure-corrected) |

## Key Methods

All methods below are called internally by `__init__`; they are also available for standalone use.

### `getE(n, y)` ŌĆö Equation of time

```python
Geo.getE(n: int, y: int) -> float
```

Computes the equation of time in **minutes** using a Fourier series over the day-angle `╬│`.

| Parameter | Type | Description |
|-----------|------|-------------|
| `n` | int | Day of year |
| `y` | int | Four-digit year (used to choose 365 or 366 denominator for leap years) |

**Returns** `float` ŌĆö equation of time in minutes.

---

### `delta(n, y)` ŌĆö Solar declination

```python
Geo.delta(n: int, y: int) -> float
```

Returns the solar declination in **radians** using a Fourier series approximation (Spencer 1971 / Iqbal form):

```
╬┤ = 0.006918 ŌłÆ 0.399912┬Ęcos(╬│) + 0.070257┬Ęsin(╬│)
  ŌłÆ 0.006758┬Ęcos(2╬│) + 0.000907┬Ęsin(2╬│)
  ŌłÆ 0.002697┬Ęcos(3╬│) + 0.001480┬Ęsin(3╬│)
```

**Returns** `float` ŌĆö declination in radians.

---

### `TOA(E0, CTZ)` ŌĆö Top-of-atmosphere irradiance

```python
Geo.TOA(E0: float, CTZ: float) -> float
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `E0` | float | Orbital eccentricity correction factor |
| `CTZ` | float | Cosine of the solar zenith angle |

**Returns** `float` ŌĆö `0` if `CTZ < 0` (sun below the horizon); otherwise `1361 ├Ś E0 ├Ś CTZ` W/m┬▓.

---

### `Mak(CTZ, TZ)` ŌĆö Kasten air mass (pressure-corrected)

```python
Geo.Mak(CTZ: float, TZ: float) -> float
```

Applies the Kasten & Young (1989) formula corrected for site pressure:

```
P       = 101355 ├Ś (288.15 / (288.15 ŌłÆ 0.0065 ├Ś alt))^(ŌłÆ5.255877)
Amk     = 1 / (CTZ + 0.15 ├Ś (93.885 ŌłÆ TZ_deg)^(ŌłÆ1.253))
Mak     = Amk ├Ś (P / 101355)
```

where `TZ_deg` is the solar zenith angle in degrees (note: the source passes `TZ` in radians as received from `math.acos`, but the formula's `93.885 ŌłÆ TZ` term treats it numerically ŌĆö be aware of units when calling standalone).

| Parameter | Type | Description |
|-----------|------|-------------|
| `CTZ` | float | Cosine of the solar zenith angle |
| `TZ` | float | Solar zenith angle in radians (from `math.acos(CTZ)`) |

**Returns** `float` ŌĆö pressure-corrected Kasten air mass.

---

### `generateGHIargp(TOA, AM)` ŌĆö ARGP clear-sky model

```python
Geo.generateGHIargp(TOA: float, AM: float) -> float
```

Computes clear-sky GHI using the ARGP (Argentine model):

```
GHI_argp = TOA ├Ś ktrp^(AM^0.678)
```

where `ktrp` is selected by `getKtrp()`:

| Condition | Formula |
|-----------|---------|
| `alt > 1000 m` | `ktrp = 0.7 + 1.6391├Ś10Ōü╗┬│ ├Ś alt^0.55` |
| `alt Ōēż 1000 m` | `ktrp = 0.7570 + 1.0112├Ś10Ōü╗ŌüĄ ├Ś alt^1.1067` |

**Returns** `float` ŌĆö clear-sky GHI in W/m┬▓ (`0` on any arithmetic exception, e.g. when `TOA = 0`).

## Usage Example

```python
import pandas as pd
from helpers.Geo import Geo

ranges = pd.date_range(
    start='2020/01/01 00:00',
    end='2023/08/31 23:59',
    freq='1min'
)

dfGeo = Geo(
    range_dates=ranges,
    lat=-22.103936,
    long=-65.599923,
    gmt=0,
    alt=3500,
    beta=0
).df

print(dfGeo[['date', 'SZA', 'TOA', 'CTZ', 'GHIargp']].head())
```

Expected output (first five rows during a pre-sunrise period):

```
                 date        SZA  TOA  CTZ  GHIargp
0 2020-01-01 00:00:00  99.195...  0.0  ...      0.0
1 2020-01-01 00:01:00  99.181...  0.0  ...      0.0
...
```

<Tip>
  For sites in UTCŌłÆ3 (Argentina), pass `gmt=-3`. The constructor automatically applies the correct sign convention so that solar hours align with local noon.
</Tip>
